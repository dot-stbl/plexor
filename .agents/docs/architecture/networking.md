# Networking — WireGuard mesh, VXLAN overlay, internal DNS, zero-touch node join

Plexor **built-in networking** для multi-node deployments:

1. **WireGuard mesh** — encrypted full-mesh VPN между control plane и всеми compute nodes (NAT-traversing)
2. **VXLAN overlay** — L2 connectivity между VMs на разных nodes (через OVS)
3. **Internal DNS** — auto-registration of nodes, VMs, app instances в `*.plexor.internal` зону
4. **Zero-touch node join** — `plx node install <control-url>` → node автоматически присоединяется к mesh, DNS, VXLAN

Без всего этого — multi-node setup ручной (настраивать WireGuard / VXLAN / DNS / firewall вручную на каждом server). С этим — один command, остальное auto.

---

## TL;DR — полная картина

```
┌────────────────────────────────────────────────────────────────────┐
│  Server A (control plane + node agent)                           │
│                                                                    │
│  Plexor.Host + Plexor.NodeAgent                                    │
│    ├─ wg0 (WireGuard): 10.200.0.1/32                            │
│    ├─ br-plexor (OVS bridge): 10.100.0.1/24                      │
│    └─ CoreDNS (internal): listens on 10.200.0.1:53                │
│                                                                    │
│  Plexor public endpoint: plexor.example.com:443 (TLS)              │
│  Plexor UI: https://plexor.example.com                             │
└────────────────────────────────────────────┬───────────────────────┘
                                             │
                          WireGuard mesh (UDP hole-punched)
                                             │
┌────────────────────────────────────────────┼───────────────────────┐
│  Server B (compute node, joined later)    │                       │
│                                            │                       │
│  Plexor.NodeAgent                           │                       │
│    ├─ wg0 (WireGuard): 10.200.0.2/32     │                       │
│    ├─ br-plexor (OVS bridge): 10.100.0.2/24                      │
│    └─ DNS client (points to 10.200.0.1:53)                        │
│                                            │                       │
│  Joined via: plx node install https://plexor.example.com           │
└────────────────────────────────────────────────────────────────────┘
```

VMs from different nodes can reach each other as if on same L2:
- VM A on Server A (10.100.0.10) pings VM B on Server B (10.100.0.20) — works through VXLAN tunnel
- VM A resolves `node-b.plexor.internal` via internal DNS → 10.200.0.2 (WireGuard IP)
- VM A resolves `vm-on-b.plexor.internal` → 10.100.0.20 (overlay IP)

## 1. WireGuard mesh (auto-VPN между nodes)

### Зачем

- **Security**: все node-to-control и node-to-node traffic зашифрован
- **NAT traversal**: WireGuard использует UDP — проходит через 99% NAT/firewall (vs IPSec/TCP)
- **Performance**: kernel module, near-wire-speed, low CPU
- **Auto-mesh**: каждый node знает peer public keys через control plane, не надо настраивать руками

### Address plan

| Subnet | Purpose | Example |
|---|---|---|
| `10.200.0.0/16` | WireGuard mesh (node-to-node) | Server A: `10.200.0.1`, Server B: `10.200.0.2` |
| `10.100.0.0/16` | VM overlay (L2 VXLAN) | VM-1: `10.100.0.10`, VM-2: `10.100.0.20` |
| `10.0.0.0/8` | Reserved for future (external VPN, public IPs, etc.) | — |

### Как работает

```
Server A (control plane):
  $ wg genkey | tee /etc/plexor/wg-private.key | wg pubkey > /etc/plexor/wg-public.key
  Public key: aAbBcC...

Server B (joining):
  $ plx node install https://plexor.example.com --token=<join-token>
    → generates local keypair
    → POSTs /admin/nodes/register with: { hostname, publicKey, listenAddr }
    → gets back: cluster config with all peer publicKeys + IP allocation
    → applies wg-quick config: /etc/wireguard/wg0.conf
    → wg-quick up wg0
    → online in Plexor dashboard

Plexor.Host:
  $ POST /admin/nodes/register
    → validates token
    → allocates next free IP from 10.200.0.0/16
    → adds node to `nodes` table
    → returns cluster config (all peer public keys + DNS config)
  $ health-check loop polls each node every 30s
```

### WireGuard config (generated by Plexor)

```ini
# /etc/wireguard/wg0.conf on Server B (after join)
[Interface]
Address = 10.200.0.2/32
PrivateKey = <local-private-key>
ListenPort = 51820

[Peer]
# Server A (control plane)
PublicKey = <server-A-public-key>
Endpoint = plexor.example.com:51820
AllowedIPs = 10.200.0.0/16, 10.100.0.0/16
PersistentKeepalive = 25

[Peer]
# Server C (other compute, if exists)
PublicKey = <server-C-public-key>
Endpoint = server-c.lan:51820
AllowedIPs = 10.200.0.3/32
```

### State (firewall-aware)

Plexor uses `iptables -A INPUT -p udp --dport 51820 -j ACCEPT` (or nftables equivalent)
during `plx node install`. If host firewall blocks UDP 51820, Plexor
falls back to **TCP tunneling** (slower but works through strict firewalls).

## 2. VXLAN overlay (L2 connectivity между nodes)

### Зачем

VMs на разных nodes должны общаться как будто в одной L2 сети
(тот же broadcast domain, ARP работает, etc.). Это нужно для:

- Кластерные workloads (k8s, Galera, etc.) — ожидают L2 connectivity
- Live migration — VM может переехать на другой node без смены IP
- Service discovery через mDNS / Avahi / Consul

### Реализация

VXLAN туннель через OVS bridge (выбран как install provider):

```
Each NodeAgent creates:
  - OVS bridge: br-plexor
  - VXLAN port: vxlan-plexor (remote=<control-plane-ip>)
  - VM tap interfaces attached to br-plexor

Plexor.Host assigns overlay IP (10.100.0.0/16) per VM
when creating VM:
  - VM-A on node-1: tap-0 → br-plexor
  - VM-B on node-2: tap-0 → br-plexor
  - Bridge uses VXLAN to forward to remote node
```

### Config (auto-generated by Plexor.NodeAgent on `plx node install`)

```bash
# /usr/local/bin/plexor-overlay-setup.sh
#!/bin/bash
set -euo pipefail

# 1. Load OVS kernel module
modprobe openvswitch

# 2. Create bridge
ovs-vsctl --may-exist add-br br-plexor

# 3. Add VXLAN port to bridge (tunnel to control plane)
ovs-vsctl --may-exist add-port br-plexor vxlan-plexor \
  -- set interface vxlan-plexor type=vxlan \
    options:remote_ip=<control-plane-public-ip> \
    options:key=plexor \
    options:dst_port=4789

# 4. Set up internal IP on bridge
ip addr add 10.100.0.<local-node-id>/24 dev br-plexor
ip link set br-plexor up

# 5. iptables: allow VXLAN
iptables -A INPUT -p udp --dport 4789 -j ACCEPT
```

### VM integration

When user creates a VM:
1. Plexor.Host allocates `10.100.0.<next>` from overlay pool
2. Control plane sends NATS event to NodeAgent: `vm.create {ip: 10.100.0.X}`
3. NodeAgent runs `plx-vm-create` which:
   - Creates tap interface
   - Attaches to `br-plexor` via OVS
   - VM gets overlay IP via cloud-init
4. VM can ping any other VM in `10.100.0.0/16` (regardless of node)

## 3. Internal DNS (CoreDNS)

### Зачем

- VMs и app instances резолвят друг друга по имени, не по IP
- `vm-{id}.plexor.internal` вместо `10.100.0.10`
- `node-XX.plexor.internal` для node-to-node communication
- `instance-{id}.{provider}.plexor.internal` для app instances

### Implementation

Plexor.Host runs **CoreDNS** (single binary, well-tested) as a sidecar
in the same process group. Configuration is generated dynamically
from the database state.

```corefile
# /etc/coredns/Corefile (auto-generated)
plexor.internal:53 {
  forward . 127.0.0.1:5353   # forward to Plexor's internal resolver
}

.:53 {
  forward . 1.1.1.1          # external queries
  cache 30
}
```

### Auto-registration

When a new resource is created, Plexor adds a DNS A record:

| Event | DNS record added |
|---|---|
| Node joins | `node-XX.plexor.internal → 10.200.0.X` (WireGuard IP) + `10.100.0.X` (overlay) |
| VM created | `vm-{id}.plexor.internal → 10.100.0.X` |
| App instance deployed | `instance-{id}.{provider}.plexor.internal → 10.100.0.X` + `instance-{id}.{provider}.public.plexor.internal → <public-ip>` (if exposed) |
| Volume attached | (no DNS — iSCSI/NFS uses IP directly) |
| LB created | `lb-{id}.plexor.internal → 10.100.0.X` |

### External DNS (Phase 2)

Plexor can integrate with external DNS (Route53, Cloudflare, PowerDNS)
for public-facing app instances. Users configure their domain
delegation to Plexor's nameservers. Plexor pushes records via API.

For MVP: external DNS is user-managed (manually add A records).

## 4. Zero-touch node join

### User-facing flow

```bash
# On Server B (new compute node)
$ plx node install https://plexor.example.com --token=abc123
[plexor] Detecting hardware (CPU, RAM, disk)...
[plexor] Generating WireGuard keypair...
[plexor] Calling control plane for join config...
[plexor] Configuring OVS bridge + VXLAN overlay...
[plexor] Setting up WireGuard interface wg0...
[plexor] Registering DNS A record (node-b.plexor.internal)...
[plexor] Starting Plexor.NodeAgent...
[plexor] Health check OK
[plexor] ✓ Node 'node-b' online (10.200.0.2 / 10.100.0.2)

# Verify from control plane
$ plx node list
NAME      IP             STATUS    CPU   MEM   DISK   UPTIME
node-a    10.200.0.1     online    8     32GB  500GB  5d
node-b    10.200.0.2     online    4     16GB  250GB  12s
```

### Implementation: `plx node install` on new server

```bash
# /opt/plexor/bin/plx-node-install.sh
#!/bin/bash
set -euo pipefail

CONTROL_URL="$1"
TOKEN="$2"

# 1. Detect hardware
CPU=$(nproc)
MEM=$(free -g | awk '/^Mem:/{print $2}')
DISK=$(df -BG / | awk 'NR==2{substr($2,1,length($2)-1)}')

# 2. Install Plexor.NodeAgent
apt-get update
apt-get install -y plexor-node-agent wireguard-tools openvswitch-switch

# 3. Generate WireGuard keypair
wg genkey | tee /etc/plexor/wg-private.key | wg pubkey > /etc/plexor/wg-public.key
HOSTNAME=$(hostname)
WG_PUBLIC=$(cat /etc/plexor/wg-public.key)

# 4. Phone home: register with control plane
REGISTER_RESPONSE=$(curl -sS -X POST \
  "$CONTROL_URL/api/v1/admin/nodes/register" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"hostname\": \"$HOSTNAME\",
    \"publicKey\": \"$WG_PUBLIC\",
    \"listenAddr\": \"<this-server-public-ip>\",
    \"hardware\": { \"cpu\": $CPU, \"memoryGb\": $MEM, \"diskGb\": $DISK }
  }")

NODE_IP=$(echo "$REGISTER_RESPONSE" | jq -r '.nodeIp')
OVERLAY_IP=$(echo "$REGISTER_RESPONSE" | jq -r '.overlayIp')
PEERS=$(echo "$REGISTER_RESPONSE" | jq -r '.peers')
DNS_IP=$(echo "$REGISTER_RESPONSE" | jq -r '.dnsIp')

# 5. Write WireGuard config
cat > /etc/wireguard/wg0.conf <<EOF
[Interface]
Address = $NODE_IP/32
PrivateKey = $(cat /etc/plexor/wg-private.key)
ListenPort = 51820
EOF

for peer in $(echo "$PEERS" | jq -c '.[]'); do
  cat >> /etc/wireguard/wg0.conf <<EOF

[Peer]
PublicKey = $(echo $peer | jq -r '.publicKey')
Endpoint = $(echo $peer | jq -r '.endpoint')
AllowedIPs = $(echo $peer | jq -r '.allowedIps')
PersistentKeepalive = 25
EOF
done

# 6. Bring up WireGuard
wg-quick up wg0

# 7. Set up VXLAN overlay
/opt/plexor/bin/plexor-overlay-setup.sh "$DNS_IP"

# 8. Start NodeAgent
systemctl enable --now plexor-node-agent

# 9. Health check
sleep 3
curl -sS "$CONTROL_URL/api/v1/admin/nodes/$(hostname)/health"
```

### Implementation: Plexor.Host side `POST /api/v1/admin/nodes/register`

```csharp
public sealed class RegisterNodeHandler(
    INodeRepository nodes,
    IClusterConfigService config,
    IDnsService dns) : ICommandHandler<RegisterNodeCommand, RegisterNodeResponse>
{
    public async ValueTask<RegisterNodeResponse> HandleAsync(
        RegisterNodeCommand cmd, CancellationToken ct)
    {
        // 1. Validate token (one-time use, expires in 1h)
        var joinToken = await tokens.ValidateAndConsume(cmd.Token, ct);
        if (joinToken is null) return new InvalidToken();

        // 2. Allocate next available IP
        var nodeIp = await config.AllocateNextNodeIp(ct);
        var overlayIp = await config.AllocateNextOverlayIp(ct);

        // 3. Persist node
        var node = new Node
        {
            Id = Guid.NewGuid(),
            Hostname = cmd.Hostname,
            PublicKey = cmd.PublicKey,
            ListenAddr = cmd.ListenAddr,
            NodeIp = nodeIp,
            OverlayIp = overlayIp,
            Status = NodeStatus.Provisioning,
            Hardware = cmd.Hardware,
            CreatedAt = DateTimeOffset.UtcNow
        };
        await nodes.AddAsync(node, ct);

        // 4. Build peer list (all existing nodes except this one)
        var peers = await nodes.ListExceptAsync(node.Id, ct);

        // 5. Register DNS A record
        await dns.AddARecordAsync(
            $"{cmd.Hostname}.plexor.internal",
            nodeIp,
            ct);

        // 6. Trigger provision transition (Provisioning → Online)
        await config.TransitionToOnlineAsync(node.Id, ct);

        return new RegisterNodeResponse(nodeIp, overlayIp, peers, dnsIp: "10.200.0.1");
    }
}
```

## 5. Failure modes & recovery

### Node loses WireGuard connection

- PersistentKeepalive = 25s detects dead peer within ~75s
- Node marked `Offline` in Plexor dashboard
- VMs on that node keep running (WireGuard is for control plane, not data plane)
- Reconnect when network restored: wg-quick auto-reconnects via handshake

### Node loses power / disk failure

- VMs on that node are `unreachable` (overlay VXLAN broken)
- Plexor marks VMs as `NodeUnavailable`
- Plexor attempts to live-migrate VMs to other nodes (if shared storage)
- If migration impossible, VMs stay in `Failed` state until node recovers

### Control plane unreachable

- Nodes still run their VMs (Plexor.NodeAgent doesn't need control plane for VM lifecycle)
- Health checks fail, node marked `Offline`
- When control plane recovers, nodes re-sync state via NATS catch-up

### VXLAN overlay split (split-brain)

- Each node's bridge continues working
- VMs on different sides can't reach each other (expected)
- When overlay reconnects → automatic recovery

### DNS server down (Plexor.Host)

- CoreDNS pod crashes → VMs can't resolve names
- Mitigation: Plexor.NodeAgent runs **local DNS cache** (read-through cache with TTL)
- Restart CoreDNS → cache warms again

## 6. Security

### WireGuard keys

- Generated locally on each node (`wg genkey`)
- Private key never leaves the node
- Public key sent to control plane during register
- Stored in `nodes.public_key` column (Plexor DB)

### Token for node registration

- One-time use join token
- Generated by Plexor admin: `plx node token create --ttl=1h`
- Expires after 1 hour (configurable)
- Logged in audit trail

### mTLS for control plane API

- All node → control plane communication over HTTPS + mTLS
- Node has its own client cert (signed by cluster CA)
- Cert rotation every 90 days (auto via Plexor.NodeAgent)

### VXLAN security

- VXLAN traffic is encapsulated in WireGuard (encrypted end-to-end)
- No additional encryption layer needed (WireGuard covers everything)

### DNS security

- Internal DNS (`*.plexor.internal`) is **only served on internal IPs** (10.200.0.0/16)
- NOT exposed externally (firewall rule)
- External queries use 1.1.1.1 upstream (not Plexor)

## 7. Configuration

### plexor.yaml (on control plane)

```yaml
cluster:
  name: company-prod
  domain: plexor.example.com
  publicEndpoint: https://plexor.example.com

networking:
  mesh:
    enabled: true
    subnet: 10.200.0.0/16
    listenPort: 51820
    tcpFallback: true                # if UDP 51820 blocked
    tcpFallbackPort: 443
  overlay:
    enabled: true
    subnet: 10.100.0.0/16
    bridge: br-plexor
    vxlanPort: 4789
  dns:
    enabled: true
    domain: plexor.internal
    upstreamDns: 1.1.1.1
    ttl: 30
  autoJoin:
    enabled: true                    # zero-touch node join
    tokenTtl: 1h
    autoApprove: false               # admin approves new nodes
```

### plexor.yaml (on compute node, after `plx node install`)

```yaml
# /etc/plexor/node.yaml
node:
  id: <auto-generated-uuid>
  hostname: server-b
  publicIp: 203.0.113.42
  publicKey: <local-wg-public-key>

controlPlane:
  endpoint: https://plexor.example.com
  mtlsCert: /etc/plexor/client.crt
  mtlsKey: /etc/plexor/client.key
  caCert: /etc/plexor/ca.crt

mesh:
  interface: wg0
  address: 10.200.0.2/32
  port: 51820

overlay:
  bridge: br-plexor
  address: 10.100.0.2/24
```
